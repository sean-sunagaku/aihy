# ADR-003: 探索型アプローチ（検証型ではなく）

- Date: 2026-04-23
- Status: Accepted

## 背景
最初は「課題 / 仮説 / 成功基準」という検証型の `problem.md` を考えたが、「そもそも課題設定が合っているか自体わからない」状態で始めることが多い。固定された成功基準では探索が妨げられる。

## 決定
`problem.md` を探索型「このアプリは何 / 誰に使ってほしい / 知りたいこと」+ 任意の「ベット」（仮置き目標、外れてもOK、当否自体が学び）で構成。

## 影響
- `feedback.md` の観点から「課題適合度」を削除、「ペルソナの主観体験」を先頭に
- 「当初想定とのズレ」観点でベットの当否・課題認識のズレを記録
- 初期ユーザーが問題を固定せずに始められる
