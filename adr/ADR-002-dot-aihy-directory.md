# ADR-002: 対象リポに `.aihy/` を配置する分散型アーキテクチャ

- Date: 2026-04-23
- Status: Accepted

## 背景
最初はフレームワーク側に `targets/<name>/` を集約する案だったが、対象アプリごとに設定を書くのは取り回しが悪く、`npx` 1コマンドで配置したい要望があった。

## 決定
フレームワーク本体は最小コードのみ持ち、対象リポごとに `.aihy/` を生成して設定・ペルソナ・履歴を全部その中に格納する分散型。

## 影響
- フレームワーク本体は `LOOP.md` / `README.md` / `init.mjs` / `package.json` のみ
- 対象リポの初期化は `npx @sean-sunagaku/aihy` 1 コマンドで完結
- 履歴は対象リポ側に集約、関心の分離が明確に
